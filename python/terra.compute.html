Loads the compute’s backend’s base module, given either a fully qualified compute backend name, or a partial (terra.compute.{partial}), and then returns a connection to the backend

  • Service based off of terra.compute.base.BaseService

terra.compute.utils.pathlib_map(volume_map, container_platform='linux')¶

Translate volume mapping from strings to pathlib objects. Source paths on the host system are instantiated as Concrete paths allowing use of the pathlib.Path.resolve() function. Paths in the container are instantiated as Pure paths of the appropriate type.

Parameters

  • volume_map (list of tuple of str) – List of tuples. Each tuple contains two strings of the form (host_str, container_str).

  • container_platform (str, optional) – String specifying container platform, linux or windows. Defaults to linux

Returns

List of tuples. Each tuple contains two pathlib objects of the form (host_pathlib, container_pathlib). host_pathlib objects are Concrete paths suitable for the current host OS. container_pathlib objects are Pure paths suitable for the specified container_platform.

Return type

list of tuple of pathlib objects
